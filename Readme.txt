AccountingApp

Проект был создан для автоматизации работы:

Администратора справочника департаментов: Ведение в актуальном состоянии
справочника департаментов

Администратора справочника работников: Ведение данных о работниках на
предприятии.

Табельщик просмотр информации о департаментах и сотрудниках работающих в них.

Список функции:

1. Вывод информации из базы данных(далее БД) на экран пользователя;
2. Добавление новой информации в БД;
3. Изменение существующей информации в БД;
4. Удаление информации из БД.

Структура проекта:

1. Папка Forms - содержит в себе средства для отображения пользовательского интерфейса;
2. Папка Class - содержит классы для работы с базой данных и диалоговыми окнами;
3. ConnectionConfiguration.config - конфигурационный файл содержажщий строку подключения к БД.

Первоначальная настройка:

1. Откройте SQL Server Management Studio и подключитесь к серверу, на котором будет хранится база данных;
2. Щелкните правой кнопкой мыши на папке "Базы данных" и выберите "Восстановление базы данных";
3. В появившемся окне выберите "Устройство" в качестве источника данных и нажмите кнопку "Обзор" рядом с ней;
4. Выберите файл резервной копии базы данных (находится в папке приложения по пути "AccountingApp\DataBase\TabelUcheta.bak") и нажмите "ОК";
5. Проверьте, что имя базы данных в поле "Восстановить как" соответствует имени базы данных, которую вы хотите восстановить;
6. В разделе "Опции восстановления" выберите опции, которые соответствуют вашим потребностям. Обычно вы можете оставить значения по умолчанию;
7. Нажмите "ОК", чтобы начать процесс восстановления;
8. Дождитесь завершения процесса восстановления базы данных. Это может занять несколько минут, в зависимости от размера базы данных и производительности сервера.
9. Получите строку подключения к вашему серверу у технического специалиста;
10. В корневой папке проекта откройте СonnectionConfiguration.config.
11. Вставьте вашу строку подключения в connectionString="Data Source=ваша строка подключения"

Руководство пользователя:

Двойным кликом по ярлыку запустите приложения. В появившемся окне авторизации введите логин и пароль полученные от администратора БД. В зависимости от того какие логин и
пароль вы получили при нажатии кнопки "Вход" откроется окно для работы на котором отображены записи из базы данных. 

Администратор справочников работников может добавлять, измененять, удалять
и просматривать информацию о работниках. Пошагово описание функций:
1.Для добавления записи в БД: в левой части открывшегося окна заполните соответсвующие поля и нажмите на кнопку Добавить, после чего откроется диалоговое окно с подтверждением действия
при подтверждении запись добавится в базу данных и обновленная таблица отобразится на экране пользователя.
2.Для изменения записи в БД: необходимо сделать двойной клик по строке в которой записана информация о сотруднике после чего поля для ввода заполнятся этой информацией которую пользователь может редактировать,
нажав на кнопку "Редактировать" информация в базе данных обновится на данные введенные пользователем.
3.Для удаления записи из БД: двойным кликом выбрать необходимую запись, нажать кнопку "Удалить", после чего откроется диалоговое окно с подтверждением действия
при подтверждении запись удалится из базы данных и обновленная таблица отобразится на экране пользователя.
4. Для поиска по базе данных: необходимо ввести параметры поиска (например "Сид" - отобразит поля со словами содержащми "Сид") и нажать на кнопку "Поиск" после чего таблица обновится и на ней отобразятся 
результаты поискаю
5. Для возврата на окно авторизации необходимо нажать на кнопку вернуться.

Над таблицей с информацией из БД находится кнопка сброса - при нажатии на неё очищаются поля для ввода и таблица с информацией возвращается стандартный вид.

Администратор справочников департаментов - смотри "Администратор справочников работников".

Табельщик двойным кликом по названию департамента в левой части окна может отобразить на экране сотрудников работающих в выбранном департаменте из базы данных.
